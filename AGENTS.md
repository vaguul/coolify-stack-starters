# Agent Guide

- Keep examples generic, safe, and useful for public self-hosting starters.
- Do not add real `.env` files, passwords, API keys, database dumps, VPS credentials, or private hostnames.
- Prefer clear health checks, named volumes, and explicit port boundaries.
- Update README, CONTRIBUTING, and workflow validation when starters change.
- Validate every compose file with `docker compose ... config` before committing.
