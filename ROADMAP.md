# Roadmap

This roadmap keeps maintenance work visible without turning the starters into a large framework.

## Current focus

- Keep every starter small, readable, and valid with `docker compose config`.
- Keep the validation matrix current when starter services, healthchecks, ports, or named volumes change.
- Keep sample credentials clearly marked as placeholders.
- Add only stacks that are common enough to maintain.

## Good first issues

- Improve comments around environment variables and internal service names.
- Add one small starter when it has a clear deployment use case and validation path.
- Add Coolify-specific deployment notes for one existing starter.

## Later

- Add optional Coolify-specific notes for domains, previews, and persistent volumes.
- Add examples for backup/restore checks where the backing service makes sense.
